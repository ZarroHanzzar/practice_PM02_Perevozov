## 4. Детальный разбор проблем

### Проблема 1: Медленное скачивание WAL из S3
- **Время:** +25 мин к этапу 2
- **Причина:** 
  - Инстанс для восстановления был типа `t2.medium` (лимит сети 100 Mbps)
  - Размер WAL за последний час = 15 ГБ
  - Скорость скачивания: 20 МБ/с вместо ожидаемых 80 МБ/с
- **Решение:**
  1. Использовать `r5.large` инстансы для восстановления (1 Гбит/с)
  2. Хранить последние 3 часа WAL в EBS gp3 с 3000 IOPS (быстрый доступ)
  3. Внедрить параллельное скачивание WAL-файлов

### Проблема 2: Ручной сбор данных с POS-терминалов
- **Время:** +35 мин к этапу 4
- **Причина:**
  - 15% магазинов были в офлайн-режиме
  - Скрипт не умел ждать и повторять попытки подключения
  - Отсутствовал механизм обратного туннеля
- **Решение:**
  1. Внедрить `reverse-ssh` туннель на всех POS-терминалах
  2. Написать скрипт с экспоненциальной задержкой (retry с backoff)
  3. Создать очередь сообщений (SQS) для автономных магазинов

### Проблема 3: Задержка переключения DNS
- **Время:** +10 мин к этапу 6
- **Причина:** TTL был установлен в 300 секунд
- **Решение:**
  - Снизить TTL для критичных A-записей до 60 секунд
  - Использовать Route53 Health Checks для автоматического failover
